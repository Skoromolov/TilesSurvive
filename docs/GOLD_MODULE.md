# Логика модуля золотодобычи (gold.py)

## Обзор

Модуль автоматизирует событие "Золотодобыча" в игре. Запускается раз в час из режима лечения.

---

## Конфигурация

В файле `config.py`:

```python
# Настройки золотодобычи
GOLD_ENABLED = True           # Включить автоматизацию
GOLD_INTERVAL = 3600          # Интервал в секундах (1 час)
GOLD_SEARCH_TIMEOUT = 60      # Таймаут поиска рудника (сек)
```

---

## Процесс работы

### 1. Проверка таймера

```
Режим HEAL
    ↓
Проверка GOLD_ENABLED
    ↓
Проверка should_do_gold() — прошло ли GOLD_INTERVAL секунд?
    ↓ Да
Переключение в режим GOLD
```

### 2. Основной процесс (`process_gold()`)

```
Шаг 3.1: Переход к Events
    ↓
navigate_to_events() — клик по events.png
    ↓
Шаг 3.2: Навигация к руднику
    ↓
navigate_to_rudnik() — клик по rudnik.png
    ↓
Шаг 3.2: Проверка открытия
    ↓
check_rudnik_opened() — проверить rudnik_opened.png
    ↓
Шаг 3.3: Поиск кнопки Find
    ↓
click_find_button() — клик по find.png
    ↓
────────────────────────────────────
Проверка результата:
    ↓
Нашли grind.png? — Да → Ветка 1
    ↓ Нет
Нашли my_rudnik.png? — Да → Ветка 2
```

### 3. Ветка 1: Добыча золота

```
Нашли find → проверили grind
    ↓
do_grind_workflow():
    ↓
Шаг 1: Клик по grind.png
    ↓
Шаг 2: Клик по work.png
    ↓
Шаг 3: Клик по go.png
    ↓
✓ Золотодобыча завершена
    ↓
update_gold_time() — обновить таймер
    ↓
Возврат к лечению
```

### 4. Ветка 2: Отзыв своего рудника

```
Не нашли grind → проверяем my_rudnik
    ↓
handle_my_rudnik_branch():
    ↓
Нашли my_rudnik.png? — Да
    ↓
click_return_button() — клик по return.png
    ↓
Возврат к проверке find (новый цикл)
    ↓
────────────────────────────────────
Если не нашли my_rudnik:
    ↓
Продолжаем поиск (новый цикл)
```

### 5. Цикл поиска

```
Максимум попыток: GOLD_SEARCH_TIMEOUT / 2 ≈ 30 раз

Цикл:
    1. Клик по find.png
    2. Проверка grind.png или my_rudnik.png
    3. Если нашли → ветка 1 или 2
    4. Если не нашли → повторить
    5. Если > 5 попыток → ошибка, возврат к лечению
```

| `gold_in_progress` | bool | Флаг выполнения процесса золотодобычи |

## Функции модуля

### `should_do_gold()`
Проверить, пора ли идти на рудник.
- **Возвращает**: True если прошло `GOLD_INTERVAL` секунд

### `update_gold_time()`
Обновить время последнего посещения рудника.

### `navigate_to_events(screen_cv, region)`
Перейти из поселения в Events.
- **Возвращает**: True если успешно

### `navigate_to_rudnik(screen_cv, region)`
Навигация к руднику в Events.
- **Возвращает**: True если успешно

### `check_rudnik_opened(screen_cv, region)`
Проверить что рудник открыт.
- **Возвращает**: True если открыт

### `click_find_button(screen_cv, region)`
Нажать кнопку поиска (find).
- **Возвращает**: True если найдена и нажата

### `check_my_rudnik(screen_cv, region)`
Проверить наличие my_rudnik.png.
- **Возвращает**: True если найден

### `click_return_button(screen_cv, region)`
Нажать кнопку отозвать (return).
- **Возвращает**: True если найдена и нажата

### `do_grind_workflow(screen_cv, region)`
Ветка 1: grind → work → go.
- **Возвращает**: True если весь процесс выполнен успешно

### `handle_my_rudnik_branch(screen_cv, region)`
Ветка 2: Проверка my_rudnik и return.
- **Возвращает**: True если удалось вернуться к find

### `process_gold(screen_cv, region)`
Основной процесс золотодобычи.
- **Возвращает**: True если процесс завершён успешно

### `finish_gold(screen_cv, region)`
Вернуться из рудника в поселение.
- **Возвращает**: True если успешно

## Интеграция в main.py

### Проверка в режиме HEAL

```python
# В режиме HEAL (автопереключение)
if GOLD_ENABLED and not gold_in_progress and should_do_gold():
    print("[MAIN] Переключение в режим GOLD")
    current_mode = MainMode.GOLD
    screen_cv = take_screenshot(window, region)
    success = process_gold(screen_cv, region)
    if success:
        print("[MAIN] Возврат к лечению после золотодобычи")
        current_mode = MainMode.HEAL
    continue
```

### Приоритеты

1. **Reconnect** — обрабатывается всегда
2. **Рейды** — если найдена кнопка присоединения
3. **Золото** — если прошло GOLD_INTERVAL и нет активного рейда
4. **Лечение** — основной режим

## Ошибки

### "Кнопка Events не найдена"
- Проверьте что вы в поселении
- Убедитесь что `events.png` правильный

### "Рудник не открыт"
- Измените `rudnik_opened.png` на другой элемент рудника

### "Превышено количество попыток"
- Увеличьте `GOLD_SEARCH_TIMEOUT` в config.py
- Проверьте правильность `find.png` и `grind.png`
