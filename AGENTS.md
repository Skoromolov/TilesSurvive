# TileSurvive — правила работы для ассистента

## 1. Кто запускает проект
- Запускать бота (`main.py`) и любые скрипты автоматизации может ТОЛЬКО пользователь.
- LLM/ассистент НЕ запускает проект, НЕ нажимает «Play» в IDE, НЕ стартует фоновые процессы бота самостоятельно.
- Ассистент может:
  - править код (`patch`, `write_file`);
  - запускать компиляцию/проверку (`python -m py_compile`, `python -c "import config"`);
  - анализировать логи, скриншоты, шаблоны;
  - давать пользователю точную команду для запуска и параметры (интерпретатор, флаги `-u`, `-B`);
  - после запуска пользователем — читать свежие логи/скриншоты и предлагать правки.

## 2. Обязательные скилы для этого проекта
Перед любой работой загрузи соответствующий скил через `skill_view(name)` и следуй его инструкциям:

- `tile-survive-bluestacks-workflow` — безопасный жизненный цикл бота (запуск, скриншот, диагностика, остановка, `__pycache__`, ловушки логирования).
- `screen-automation-bot-patterns` — паттерны стейт-машины, OpenCV, добавление режимов, типичные ловушки и пороги.
- `systematic-debugging` — 4-фазный анализ логов и ошибок.
- `bluestacks-bot-testing` — безопасный запуск/остановка фоновых процессов бота.
- `documentation-updates` — синхронизация README/docs с кодом.

## 3. Анализ логов и ошибок — алгоритм
1. Прочитать свежий `logs/bot.log` или `debug_screenshots/`, отсортированные по времени.
2. Найти первую аномалию: повторяющееся состояние, одинаковые confidence, цикл UNKNOWN→back.png, отсутствие ожидаемого перехода.
3. Проверить, не stale-ли код: старые пороги в логах (`порог=0.6/0.7`) при файлах с 0.35, отсутствие новых лог-строк = пользователь не перезапустил скрипт.
4. Если stale-код — сказать ОДИН РАЗ: «Перезапусти скрипт: Ctrl+C → python -B main.py» и прекратить повторный анализ одних и тех же логов.
5. Если новый баг — сделать скриншот `debug_screenshots/`, запустить `cv2.matchTemplate` через системный Python 3.13 (`C:/Users/defaultuser0/AppData/Local/Programs/Python/Python313/python.exe`), определить реальный confidence шаблонов.
6. Применить минимальное исправление, проверить `python -m py_compile main.py heal.py raid.py gold.py config.py utils.py adventure.py logger.py`.
7. После правки: сообщить, что именно изменено, и напомнить пользователю о необходимости перезапуска (ассистент не запускает сам).

## 4. Запрещённые действия
- Не запускать `main.py`, `gold.py`, heal/raid/adventure процессы в фоне самостоятельно.
- Не использовать `taskkill //F //IM python.exe`, `pkill -9 python` — это убьёт Hermes и IDE.
- Не нажимать Escape или горячие клавиши для закрытия попапов без явного подтверждения пользователя.
- Не придумывать UI-шорткаты, не использовать `book.png` для событий, не использовать `rudnik.png` внутри календаря.
- Не понижать глобальный `CONFIDENCE_THRESHOLD`; использовать локальные пороги и обновлять шаблоны.
- **НЕ устанавливать порог (threshold) ниже `0.7` ни в коде, ни в конфигурации.** Если шаблон не находится при `threshold >= 0.7`, это не повод понижать threshold — в 99% случаев проблема в другом (устаревший/неправильный шаблон, не тот UI-стейт, задержка анимации, stale `__pycache__`, неверная область поиска). В таком случае нужно: сделать скриншот, запустить `cv2.matchTemplate` с системным Python 3.13, определить реальный confidence, и **вызвать пользователя** с выводом-причиной. Изменение threshold разрешено только после явного апрува пользователя.

## 5. Обязательные проверки перед правкой
- Изучить документированный flow (`README.md`, `docs/`, `tmp/`, `references/` в скиллах) и соответствующий `determine_*_state`/`process_*`.
- Проверить, что константа из `config.py` существует и правильно названа.
- Проверить `init.sql`/DDL только если проект использует БД (TileSurvive — нет).
- Убедиться, что после правки не осталось `print()` вместо `logger.info()`.

## 6. Формат отчёта
- Кратко: что изменилось и в каком файле.
- Реальный результат проверки (`py_compile`, тесты, git status).
- Если нужен запуск — точная команда для пользователя, ассистент не запускает сам.
- Никаких извинений, маркдаун-заголовков-красот, лишней воды.
